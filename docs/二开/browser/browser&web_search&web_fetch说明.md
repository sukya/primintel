`browser` 工具是 Primintel 工具箱里的“重型武器”，它与之前优化的 `web_search`（搜片段）和 `web_fetch`（读正文）构成了搜索能力的完整梯队。

### 🛠️ 三位一体的定位：
*   **web_search**：快速、低成本。用于在大海里捞针（看索引和摘要）。
*   **web_fetch**：中等深度。用于直接读网页文字（看正文）。
*   **browser**：**高深度、重交互**。用于处理 JavaScript 动态加载的页面、需要点击交互的操作，或者需要“视觉验证”的场景。

---

### 🌟 您可以尝试的四个场景：

#### 1. 视觉规格验证（带截图）
如果您想确保看到的规格是 100% 准确的，或者需要看布局。
*   **指令建议**：“帮我打开 NVIDIA 官网的 RTX 4090 产品页，找到技术规格部分，给我一张清晰的规格截图，并告诉我它的建议电源瓦数是多少。”
*   **看点**：Agent 会导航、截图并利用 Vision 能力分析图片内容。

#### 2. 点击与多级导航
有些信息藏在按钮后面，或者需要跳转多次。
*   **指令建议**：“去 Github 搜索并打开 'antigravity' 仓库，点击 'Commits' 列表，告诉我最近一次提交的作者和内容是什么。”
*   **看点**：Agent 会利用 `snapshot` 识别页面上的按钮编号（ref），然后执行 `click`。

#### 3. 动态数据抓取（JS 渲染）
很多金融曲线、实时报价页面在 `web_fetch` 下是空白的。
*   **指令建议**：“去网页版行情站查看目前比特币的 24 小时交易量，并给我截取一张当前的价格走势图。”
*   **看点**：Browser 能够执行 JavaScript 渲染出动态图表，并保存截图供您查看。

#### 4. 处理简单的遮挡或弹窗
如果一个页面进入时有弹窗阻挡了视线。
*   **指令建议**：“帮我浏览一下某个特定新闻网站的头条，如果中间有干扰弹窗，请尝试关闭它或滚动页面，然后告诉我头条内容。”
*   **看点**：Agent 可以执行 `click` 关闭按钮或 `scroll` 滚动过干扰区。
